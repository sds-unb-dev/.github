# Contribuição

Obrigado por contribuir. Este documento descreve de forma breve o processo esperado para contribuir com os repositórios da organização.

1. Abra uma issue para discutir mudanças relevantes antes de implementá-las.
2. Crie uma branch com nome descritivo: `feature/`, `fix/`, `chore/`.
3. Envie uma Pull Request pequena e focada, referenciando a issue relacionada.
4. Inclua testes automatizados quando aplicável e documentação sucinta para mudanças de comportamento.
5. Siga o padrão de commits adotado pela organização.
6. Marque os revisores apropriados; PRs que afetem áreas críticas devem incluir revisão de `CODEOWNERS`.

Regras de revisão:
- Priorize clareza e segurança.
- Dê feedback construtivo e objetivo.
