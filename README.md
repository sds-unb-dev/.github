# .github — Configurações organizacionais

Este repositório centraliza arquivos e templates usados por todos os repositórios da organização Sala de Situação — UNB. Ele facilita consistência operacional e governança ao distribuir padrões, templates e automações reutilizáveis.

## Conteúdo principal

- `profile/README.md` — descrição do perfil da organização
- `ISSUE_TEMPLATE/` — modelos de issue (bug, feature, etc.)
- `PULL_REQUEST_TEMPLATE.md` — template de pull request
- `CONTRIBUTING.md` — diretrizes de contribuição organizacionais
- `CODE_OF_CONDUCT.md`, `SECURITY.md` — políticas de comunidade e segurança
- `CODEOWNERS` — proprietários de código e áreas de responsabilidade
- `dependabot.yml` — configuração de atualizações automáticas
- `workflows/` — (opcional) workflows compartilhados do GitHub Actions

## Como usar

- Repositórios individuais podem adotar estes templates e workflows copiando-os ou referenciando-os conforme necessário.
- Ative recursos de saúde da comunidade (templates de issue/PR) nas configurações do repositório quando aplicável.

## Manutenção

- Abra uma issue aqui para propor mudanças de impacto.
- Submeta Pull Requests pequenas e documentadas; marque os responsáveis em `CODEOWNERS` para revisão.
- Teste alterações significativas (ex.: workflows) em um repositório piloto antes de aplicar globalmente.

## Como contribuir

- Discuta mudanças maiores via issue antes de implementar.
- Use branches descritivas e PRs focadas; inclua testes e documentação sucinta quando necessário.
